How ARNet Discovery is used in the field

The app is designed for engineers who need quick visibility during panel FAT, commissioning, relay LAN checks, and troubleshooting. Start with a local scan, probe a known IP, or import the official engineering IP list.

1. Select adapterChoose the LAN/RJ45 adapter connected to the panel, switch, or engineering network.
2. Scan or importUse Scan Local for nearby devices, Probe for a known IP, or Import Excel for exact project targets.
3. Read evidenceReview ping, port, protocol, expected-device metadata, and diagnostics in one table.
4. Export resultExport CSV evidence for FAT records, punch-list checks, or follow-up troubleshooting.

Frequently asked questions

Is ARNet Discovery a replacement for commissioning procedures?

No. It is an engineering visibility tool. Use it to gather device and protocol evidence, then follow the project commissioning, cyber-security, and protection procedures.

Can it scan IPs outside my current subnet?

Yes, use Probe or Import Excel / Scan List. If routing exists from the laptop, ARNet can check exact target IPs outside the local safe scan window.

Why does an expected relay show “No response”?

Common causes include VLAN mismatch, no route, cable issue, device power, firewall, wrong IP, or a relay that blocks ICMP and the checked protocol ports.

Does ARNet aggressively scan the network?

No. The default profile uses bounded targets, short timeouts, and lightweight evidence checks so the UI stays responsive and the scan remains conservative.
